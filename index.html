Магазин в Telegram - Решение CORS

Устраняем проблемы с Cross-Origin Resource Sharing в Telegram Mini Apps

Решение для вашего проекта

Проблема

Ваш Telegram WebApp пытается получить доступ к API, но возникает ошибка CORS:

// Сообщение об ошибке Access to fetch at 'https://api.example.com/data' from origin 'https://your-telegram-app.com' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.

Это происходит из-за политики безопасности браузера, которая блокирует запросы с других доменов, если сервер явно не разрешил их.

Решение

Вам нужно настроить ваш сервер для включения правильных CORS заголовков в ответах.

Node.js/Express
Vercel
Nginx

Для сервера на Node.js/Express используйте пакет cors:

// Установите пакет cors // npm install cors const express = require('express'); const cors = require('cors'); const app = express(); // Включите CORS для всех маршрутов app.use(cors({ origin: ['https://your-telegram-app.com', 'https://your-vercel-app.vercel.app'] })); // Или включите CORS для конкретного маршрута app.get('/api/data', cors({ origin: 'https://your-telegram-app.com' }), (req, res) => { res.json({ message: 'Этот маршрут имеет включенный CORS' }); });

Если вы используете Vercel, настройте CORS с помощью файла vercel.json:

// vercel.json { "headers": [ { "source": "/api/(.*)", "headers": [ { "key": "Access-Control-Allow-Origin", "value": "https://your-telegram-app.com" }, { "key": "Access-Control-Allow-Methods", "value": "GET, POST, PUT, DELETE, OPTIONS" }, { "key": "Access-Control-Allow-Headers", "value": "X-Requested-With, Content-Type, Accept" } ] } ] }

Для сервера на Nginx добавьте заголовки в конфигурацию:

location /api/ { # Разрешить запросы с определенных доменов add_header 'Access-Control-Allow-Origin' 'https://your-telegram-app.com'; add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS, PUT, DELETE'; add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range'; # Обработка preflight запросов if ($request_method = 'OPTIONS') { add_header 'Access-Control-Allow-Origin' 'https://your-telegram-app.com'; add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS, PUT, DELETE'; add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Range'; add_header 'Content-Type' 'text/plain; charset=utf-8'; add_header 'Content-Length' 0; return 204; } }

Дополнительные рекомендации

1
Указывайте точные домены вместо использования звездочки (*) для лучшей безопасности
2
Обрабатывайте preflight запросы (OPTIONS) правильно на вашем сервере
3
Используйте переменные окружения для управления разрешенными доменами в разных средах
4
Для Telegram Web Apps убедитесь, что включили все возможные URL развертывания в разрешенные домены
Частая ошибка: Забыть указать протокол (https://) в разрешенных доменах.
